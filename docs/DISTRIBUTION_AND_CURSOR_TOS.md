# Distribution & Cursor Terms of Service (informational)

**Not legal advice.** If you sell or widely distribute this app, confirm compliance with Cursor’s current Terms of Service and any API or branding rules at [cursor.com](https://cursor.com).

## Typical distinction

- **Selling or sharing your app** (e.g. a Flutter client that uses **the user’s own** Cursor API key) is a different question from **reselling or redistributing Cursor’s service** without authorization.
- You remain responsible for: accurate store listings, privacy policy, security of user data, and not misleading users about who provides the AI/IDE services.

## What to disclose to users

- The app uses **Cursor’s Cloud Agents API** when configured with a user key.
- **Private AI** mode talks only to servers the user configures (e.g. local Ollama).
- **Capabilities** may call **user-defined webhooks** and a **user-run bridge** on their PC.

Point power users to Cursor’s own terms and dashboard for key management.

## Branding

Avoid implying the app is an official Cursor product unless you have permission. Use clear naming (e.g. “unofficial client” or your own product name) if required by store or Cursor guidelines.
